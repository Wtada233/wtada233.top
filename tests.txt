测试指南：生产环境

前言
本测试指南旨在验证基于 Astro 框架部署的博客在生产环境下的功能、性能和用户体验。所有测试都应在通过 `pnpm build` 命令生成的静态文件（位于 `dist` 目录）或已部署的生产网站上进行。请确保您测试的是最终的编译版本。

I. 部署与基本访问验证

1.  网站可访问性
    *   通过生产 URL 访问网站。
    *   确认网站首页能够成功加载，没有网络错误（检查浏览器控制台 Network 标签）。
    *   确认所有关键资源（CSS, JS, 图片, 字体）均已正确加载，HTTP 状态码均为 200。

2.  CDN / 静态资源 (如果适用)
    *   如果使用了 CDN，检查所有静态资源是否通过 CDN 域名加载。
    *   检查图片等资源是否被优化（例如，是否正确转换为 WebP 格式等）。

3.  浏览器控制台错误
    *   打开浏览器开发者工具，检查 Console 标签页，确认没有任何 JavaScript 错误或警告。

4.  断开链接
    *   手动点击导航菜单、文章列表中的所有链接，确认没有 404 页面或错误的重定向。
    *   (可选) 使用网站爬虫工具（如 Screaming Frog SEO Spider）检查全站是否存在任何断开的内部链接。

II. 核心功能测试

1.  首页内容
    *   确认首页文章列表正常显示，所有文章卡片包含正确的标题、描述、标签、分类、发布日期、字数和阅读时间。
    *   确认侧边栏小部件（例如：博客信息、分类、标签、运行时间、个人资料）正常显示且数据正确。
    *   确认文章的置顶/置底标记（如果配置启用）显示正确。
    *   验证侧边栏小部件中的“更多”按钮（如果存在）是否按预期工作。
    *   确认主页横幅（如果启用）及其文本正常显示。

2.  文章详情页
    *   点击任意文章进入详情页。
    *   确认文章标题、内容（包括 Markdown 渲染）、图片、代码块（高亮、行号、复制按钮）、AI 摘要（如果启用）正常显示。
    *   确认文章元数据（发布日期、更新日期、标签、分类）显示正确。
    *   确认文章内的链接（包括内部和外部链接）和图片点击（PhotoSwipe 灯箱功能）正常工作。
    *   确认文章底部的许可证信息和分享按钮（如果启用）正常显示。
    *   确认文章底部的“最后更新时间”卡片（如果启用且存在）显示正确，并正确判断是否“已过时”。
    *   确认文章底部的相关文章列表（如果启用）显示正确。
    *   确认 Giscus 评论区（如果启用）正常加载，且主题与网站主题一致。
    *   确认上下篇文章导航链接（如果存在）正常工作。

3.  导航功能
    *   点击导航栏中的所有链接（主页、归档、关于、系列、外部链接），确认跳转到正确的页面。
    *   验证 Swup 页面过渡效果是否平滑，没有闪烁或内容加载延迟。

4.  归档页 (`/archive`)
    *   确认文章按时间、系列、分类、标签分组显示。
    *   点击不同的视图模式按钮（例如：按时间、按系列、按分类、按标签），确认数据正确切换和显示。
    *   确认 URL 参数筛选器（例如 `/?tag=xxx`, `/?category=xxx`, `/?series=xxx`, `/?uncategorized=true`）正常工作，并能正确筛选文章。

5.  系列页 (`/series`)
    *   确认系列列表页正常显示，展示所有系列及其包含的文章数量。
    *   点击单个系列，确认能跳转到该系列的文章列表。

6.  分类和标签页
    *   点击侧边栏或文章元数据中的分类/标签，确认能正确筛选文章列表。

7.  搜索功能 (Pagefind)
    *   在搜索框中输入关键词，确认搜索结果准确且与内容相关。
    *   确认搜索结果链接能正确跳转到对应的文章。

8.  404 页面
    *   尝试访问一个不存在的 URL (例如 `yourdomain.com/nonexistent-page`)。
    *   确认 404 页面正确显示，并包含返回主页和归档页的链接。

III. UI/UX 与交互测试

1.  响应式设计
    *   在不同设备（桌面、平板、手机）或通过调整浏览器窗口大小，确认页面布局、图片、字体、导航栏等元素自适应良好，没有布局错乱。
    *   检查移动端导航菜单 (`nav-menu-panel`) 是否正常展开和收起。

2.  主题切换
    *   点击亮/暗模式切换按钮，确认网站主题（颜色、背景、文本颜色）能正确切换。
    *   确认“跟随系统”模式正常工作，并根据系统主题自动切换。

3.  玻璃模式 (Glass Mode)
    *   点击玻璃模式切换按钮，确认页面元素（卡片、浮动面板）背景变为半透明磨砂玻璃效果。

4.  主题色调整 (如果配置启用)
    *   拖动主题色选择器的滑块，确认主题色能实时改变。
    *   确认重置按钮能恢复默认主题色。

5.  阅读进度条
    *   在文章详情页滚动页面，确认顶部阅读进度条能动态显示阅读进度。

6.  返回顶部按钮
    *   向下滚动页面，确认“返回顶部”按钮出现。
    *   点击按钮，确认页面能平滑滚动回顶部。

7.  半透明导航栏 (Semifull Nav)
    *   如果导航栏透明模式设置为 `semifull`：
        *   在主页上滚动页面，确认导航栏在滚动一定距离后变为不透明/磨砂状态。
        *   在非主页上，确认导航栏始终为不透明/磨砂状态。

8.  动画与效果
    *   页面加载动画 (`onload-animation`)：确认组件（导航栏、侧边栏、文章列表等）加载时有淡入动画。
    *   滚动动画 (`scroll-animate`)：确认文章详情页内容滚动进入视图时有动画效果。
    *   波纹效果 (`ripple-effect`)：确认点击按钮或链接时有波纹效果。
    *   粒子效果 (`initEffects`)：确认点击和鼠标移动时有粒子效果（如果启用）。

IV. 性能测试 (基本)

1.  页面加载速度
    *   使用浏览器开发者工具（Network 标签页）或在线工具（如 Google PageSpeed Insights），检查关键页面的加载时间、首次内容绘制 (FCP) 和最大内容绘制 (LCP) 指标。
    *   确认没有大的资源加载瓶颈或加载时间过长的请求。

V. SEO 与可访问性 (基本)

1.  元数据 (Metadata)
    *   检查页面 `<head>` 中的 `title`、`description`、`keywords`、`og:title`、`twitter:title` 等标签是否正确生成且内容准确。
2.  Sitemap 和 Robots.txt
    *   访问 `yourdomain.com/sitemap-index.xml`，确认站点地图存在并包含所有可索引页面。
    *   访问 `yourdomain.com/robots.txt`，确认配置正确，没有意外的禁止抓取规则。
3.  语义化 HTML
    *   使用浏览器开发者工具或 Lighthouse 检查 HTML 结构是否语义化，没有明显的结构错误。
4.  键盘导航
    *   尝试仅使用键盘（Tab, Enter, Space）浏览网站，确认所有可交互元素都能被选中和激活。
    *   确认 Tab 键的焦点顺序逻辑正确。
